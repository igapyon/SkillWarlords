### 覇道式ソフトウェア開発者スキル対応表

| パラメータ       | 開発スキル的意味合い                                                                                                                       | 特に重要とされる<br>職種／役割                                                    | 備考                                                                                                                                                                       |
|----------------------------------|-----------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **統率（Leadership）**          | チームを統率し、進行管理を行い、適切な指示を出す力。 メンバーの状態把握からタスクの優先度調整、全体方針の決定までを担う。                                                   | プロジェクトマネージャー（PM）、<br>テックリード                                 | チームビルディング、意思決定、リソース管理、進捗管理など。炎上時には迅速な判断やコミュニケーションが求められ、チーム全体が最大成果を出せる環境を構築する責任を負う。                                          |
| **武力（Execution）**           | 実装力、作業量、スピード、修羅場対応力。コードを書く、バグを潰す、仕様を形にする、といった「手を動かし成果を出す」実行面における強さ。                                   | 実装エンジニア、<br>バグハンター、<br>現場型リードエンジニア                     | 短期間での機能実装や緊急対応、火消し役など、現場レベルで直接“武力”を発揮し続けることが期待される。スピード感や粘り強さが重要であり、高い武力は開発チームの土台を支える。                                    |
| **知力（Technical Intelligence）** | システム設計や技術選定の判断力、問題を発見して解決する力。将来の拡張性・スケーラビリティを見据え、最適なアーキテクチャや技術スタックを引き当てる“頭脳”。               | アーキテクト、<br>テックリード、<br>R&Dエンジニア                                 | 「なぜその設計や技術を選ぶのか」を論理的に説明でき、メリット・デメリットを整理する。長期的な運用コストや保守性まで考慮して判断するため、プロジェクト全体の品質に直結する。                                |
| **政治（Negotiation）**         | 要件やスケジュール、ステークホルダー間の意見調整など、社内外の利害を取りまとめる折衝力。 相反する要求をどうまとめ上げるか、顧客や各部門ともうまく合意形成する力。       | PM、<br>ディレクター、<br>営業寄りエンジニア                                     | 変更要求や予算・スケジュールの交渉など、異なる視点を持つ関係者を調整するうえで必要不可欠。プロジェクトのフェーズが進むほど政治スキルの重要度が増し、全体の潤滑油として機能する。                             |
| **魅力（Charisma）**            | 人望や信頼、空気作り、心理的安全性の確保を担う“チームの雰囲気を良くする”能力。後輩指導や教育を通じてメンバーの成長を促し、協力体制を築く。                           | メンター、<br>スクラムマスター、<br>教育係、<br>人気エンジニア                   | 周囲のモチベーションを高め、チームを一体化する潤滑油的役割。チームの結束力を強化することで離職率を下げたり、新人が早く馴染める環境を作るなど、「ソフトスキルの王様」といえるほど影響力が大きい。             |

## 戦法 （アクティブスキル）

| 戦法名 | 概要 | 発動場面・効果 |
|--------|------|----------------|
| **対話設計ファシリテーション** | 議論が並列に散らばってる状態を、段階的な問いで整理し、最終的な結論へと導く。 | 議論がグダるMTGで発動。「それってAとB、どっちの話ですか？」で流れを制圧。 |
| **リアルタイム図解化** | 話しながらMiro／Whimsical／手書きなどで構造を即座に図解し、全員の認識を同期する。 | 「誰か図にしてくれ…」って空気を、1人で回収していく神の手。 |
| **アジェンダ再構成** | 会議の中で「この順番だと進まない」と気づいた瞬間、さりげなく論点順を差し替えて流れを変える。 | PMやリードに多い“会議のドライビングスキル”の発露。地味だけど強い。 |
| **リモート沈黙破壊** | Zoom等で話題が止まった瞬間に、絶妙なトーンで空気を切り替える発話を入れる。 | 「あ、ちょっと思ったんですけど……」が全員の救いになる。 |
| **コードベース瞬間把握** | 巨大なコードでも30分で主要構造を把握して、すぐ改修作業に入る。 | 急な障害対応やレガシーコードの改修時に輝く。 |
| **デスマーチ食いしばり** | 終わらない仕様変更、詰まった設計、休日のデプロイ… それでも折れずに最後までやりきる。 | 絶望的プロジェクトの現場で「この人がいたから終わった」と言われる。 |
| **GitHubの守護者** | PRレビューでミス・ヌケ・破壊的変更を事前に防ぎ、全体の品質と一貫性を維持する。 | 「あの人がapproveしない限りマージされない」という安心感。 |
| **レアバグ可視化の陣** | 他の人が再現できなかった挙動を偶然 or 直感で見つけ、最小再現コードを作ってくる。 | 「えっ…なんで気づけたの？」と周囲が戦慄する。 |
| **爆速ホットフィックス** | 重大障害発生時に、修正・テスト・デプロイまでを最短ルートで完了。 | サービス継続のヒーロー。ログを追うスピードが異常。 |
| **カオス仕様整理爆撃** | 設計図なき機能を実装する際、暗黙知・Slackログ・旧コードをまとめて“設計らしきもの”を作る。 | 「これは設計図じゃなくて設計遺跡」と言われるような案件で神と化す。 |

## 技能 （パッシブスキル）

| 技能名 | 内容 | 活きる場面 |
|--------|------|------------|
| **視覚言語化力（グラレコ技能）** | 会話を即座に図・構造に落とし込む。流れ、関係、対立、解決策を空間的に提示できる。 | ドメイン理解、要件整理、合意形成などに強烈な効果。 |
| **論点抽出力** | 雑多な会話の中から、「論点」「意思決定すべき項目」「保留事項」を分けて提示できる。 | 進行が迷子になりがちなチームで発動すると、全員が助かる。 |
| **爆速タイピング流派** | 速度と正確性を両立し、あらゆるツールで操作効率が異常。 | 実装・リファクタ・Git操作・CLI操作すべてにおいて「速い」。 |
| **デグレ敏感体質** | UIや挙動の細かな変化にすぐ気づく。レビューでも自分の改修でもミスを見逃さない。 | QA不要説が出るレベルの完成度を維持する。 |
| **隠れ静的解析魔人** | コンパイルエラーやlinter警告が出る前に、コードを頭の中で走らせて違和感を察知。 | 設計ミスを未然に防ぎ、他人のコードでも「この変数気持ち悪くない？」と指摘してくる。 |
| **トレース耐性** | エラー時のstack traceやログを即座に読み解き、原因を特定するのが異様に速い。 | 複雑なマイクロサービスやクラウド環境でも強い。 |
| **コード構成美学** | 名前・責務・階層構造・依存の切り方に強いこだわりがあり、綺麗なコードを量産する。 | 後続の開発者から密かに尊敬されている。 |
| **闇コード通訳者** | 誰もドキュメントを書いてないコードを読み、何をしているのかを自然言語で説明できる。 | レガシー案件で極めて重要。新人にとっての希望。 |
| **不具合とのチャネラー** | 自然体で操作しているだけなのに、誰も気づかなかったレアバグ・偶発バグを引き当ててしまう特異体質。 | テスト、QA、受け入れ確認などで異常に強い威力を発揮する。再現が難しいバグの初発見者になりがち。 |
